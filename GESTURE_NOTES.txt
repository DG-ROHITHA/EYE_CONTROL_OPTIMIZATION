========================================
GESTURE VOCABULARY (11 GESTURES)
========================================

1. PINCH (thumb + index tip close)
   → MOUSE_CLICK at cursor position
   Cooldown: 500ms before next pinch

2. DOUBLE_PINCH (two pinches within 400ms)
   → MOUSE_DOUBLE_CLICK

3. OPEN_PALM_HOLD (all fingers extended, ≥1000ms)
   → SCROLL_MODE_TOGGLE
   Enables scroll control via gaze direction

4. FIST (all fingers curled)
   → CANCEL_COMMAND (drains queue immediately)
   Priority: EMERGENCY level

5. TWO_FINGER_SWIPE_UP (index + middle extended, wrist velocity up)
   → SCROLL_UP (magnitude proportional to speed)

6. TWO_FINGER_SWIPE_DOWN (same as above, velocity down)
   → SCROLL_DOWN

7. THUMB_UP (thumb extended upward)
   → CONFIRM (yes/accept)

8. THUMB_DOWN (thumb extended downward)
   → REJECT (no/cancel)

9. INDEX_POINT (index extended, others curled, ≥300ms)
   → CURSOR_OVERRIDE_START (use hand position as cursor)

10. THREE_FINGER_SPREAD (index + middle + ring extended + spread)
    → EMERGENCY_CALL_NURSE (highest priority, bypasses threshold)
    Triggers audio alert (1200 Hz beep, 500ms)

11. NEUTRAL (no specific gesture)
    → No action (allows idle state)

========================================
FUSION MODES (USER-SELECTABLE VIA F KEY)
========================================

🟢 EYE_LEADS_HAND_CONFIRMS (SAFEST - default)
   Eye gaze sets target. Hand gesture (pinch) fires command.
   Perfect for: Fatigue-prone patients, eye accuracy issues
   Benefit: Zero accidental clicks from saccades

🔵 HAND_LEADS_EYE_CONFIRMS (MEDIUM SAFETY)
   Hand position sets target. Eye dwell (300ms) fires command.
   Perfect for: Users with poor eye control, good hand mobility
   Benefit: Hands-first control flow

🟡 PARALLEL (STANDARD)
   Either eye OR hand can trigger independently.
   Perfect for: Users who switch between modalities fluidly
   Benefit: Maximum flexibility

🔴 HAND_OVERRIDE (MINIMAL SAFETY)
   Hand takes priority, eye is fallback.
   Perfect for: Excellent hand control, poor eye tracking
   Benefit: Responsive to hand first

========================================
USER WORKFLOW
========================================

FIRST RUN (New User):
1. Application starts
2. NeuroGaze Elite initializes all modules
3. Eye calibration prompt (5-point, press C)
4. System awaits hand gesture activity
5. (Optional) Start hand calibration when ready
   - Press 'H' to begin 20s passive calibration
   - Move hand naturally, keep all 5 fingers visible
   - System auto-saves profile to ~/.neurogaze/profiles/
6. System ready for eye-hand fusion control
7. Press 'F' to select fusion mode based on patient capability

ONGOING USE:
1. Gaze at desired target (eye sets cursor position)
2. Pinch to click (hand confirms action)
3. Use FIST to cancel if needed
4. Press 'F' to switch fusion modes if modality preference changes
5. Press 'G' to toggle gesture visualization for debugging

========================================
ERROR HANDLING & GRACEFUL DEGRADATION
========================================

✓ No hand model available
  → System runs eye-only (existing behavior unchanged)
  → All hand gesture features disabled silently
  → Logs warning, continues operation

✓ Hand detection fails on frame
  → Gesture classification skipped for that frame
  → Fusion engine uses last-known gesture state
  → Intent engine continues eye-only analysis

✓ DL model unavailable
  → Falls back to MediaPipe-only gaze (existing behavior)
  → Hand gestures unaffected
  → Logs warning at startup

✓ Gesture calibration poor quality (<60%)
  → System warns user
  → Prompts for recalibration
  → Falls back to default thresholds meanwhile

- Hand size normalization prevents threshold mismatches
- Per-user profiles eliminate need for re-setup after re-login
- YAML gesture mapping supports custom healthcare facility workflows

SAFETY FEATURES:
- Command deduplication (0.3s cooldown prevents rapid-fire clicks)
- Priority queue prevents CANCEL from losing EMERGENCY_ALERT
- Confidence threshold (70%) filters low-confidence predictions
- Strain monitoring prevents damage from extended sessions (20-20-20 breaks, 45min limit)
